資料安全架構
🔐
SMART on FHIR OAuth 2.0
病人須明確授權，Access Token 15 分鐘短效，搭配 Refresh Token。未授權診所無法讀取跨院資料。
🔒
TLS 1.3 全程加密
所有 FHIR API 傳輸均經 TLS 1.3 加密，禁止明文 HTTP，EV SSL 憑證每 90 天自動更新。
🛡
Patient ID 與身分資料分離
系統內部僅流通 PID，身分證號加密儲存於獨立資料庫，不隨 FHIR 資源流通。
🔍
AuditEvent 存取稽核
每筆資料讀寫均產生 FHIR AuditEvent，記錄操作者、時間、IP、動作，保存 7 年。
📋
Consent resource 授權管理
病人可透過 FHIR Consent 細粒度管理：哪間診所可讀、哪類資料可存取、有效期限設定。
🏥
Organization 隔離
每間診所有唯一 ORG ID，A 診所預設無法讀 B 診所建立的紀錄，除非病人明確授權。
🔑
meta.security 機敏標記
每筆 Observation 加 meta.security 標記，機敏資料需額外授權才能讀取。
